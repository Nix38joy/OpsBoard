# OpsBoard

OpsBoard — React-приложение для управления инцидентами, имитирующее реальный B2B-продукт для операционных команд.  
Проект сфокусирован на задачах уровня frontend middle: role-based доступ, data-heavy экраны, валидация форм, бизнес-правила и тестовое покрытие критичных сценариев.

## Что входит в MVP

- Mock-аутентификация с сохранением сессии
- Разграничение прав по ролям: `viewer`, `operator`, `admin`
- Список инцидентов: поиск, фильтры, пагинация, флаг `overdue`
- Деталка инцидента: переходы статусов, комментарии, timeline событий
- Создание и редактирование инцидентов (`React Hook Form` + `zod`)
- Dashboard с KPI-карточками и переходами в предфильтрованный список
- Критичные автотесты (доступ по роутам, валидация форм, бизнес-правила)

## Технологический стек

## Роли и права

- `viewer`
  - Только просмотр dashboard, списка инцидентов и деталей
  - Не может создавать/редактировать инциденты и работать с комментариями
- `operator`
  - Может создавать и редактировать инциденты
  - Может менять статус по разрешённым переходам
  - Может добавлять комментарии
- `admin`
  - Все права `operator`
  - Дополнительные привилегии (например reopen из `closed`)
  - Может удалять комментарии

## Ключевые бизнес-правила

- Переходы статусов учитывают роль и валидируются в data-слое
- `overdue` рассчитывается для активных инцидентов старше порога
- Добавление комментариев ограничено ролью
- Удаление комментариев доступно только `admin`
- Важные действия пишутся в timeline для audit-like прозрачности

## Структура проекта
